Installing the CLI is only half the path. This page is the product checklist: health → sessions & persona → workers → handsfree. Prefer letting the /hark skill drive questions by voice; the CLI matches the same order.

First-run checklist Install methods

Prerequisites

  • Python 3.11+ and a working mic/speakers (PortAudio; often ffmpeg for some paths).
  • Herdr ≥ 0.7.1 for the sessions you will watch.
  • Speech auth for cloud STT/TTS (xAI OAuth, OpenAI, Google, MiniMax TTS, …) — hark doctor reports what it finds.
  • Skills-only is not enough. npx skills / the npm package copy agent skills; they do not install the hark CLI, wake models, or system deps. See POST_INSTALL.md.

Install (short)

Full picker lives on the homepage Install section. Hosted script: https://hark.xk.io/install.sh (published with each version tag).

Recommended: inspect, then run

bash · inspect first
$ curl -fsSL https://hark.xk.io/install.sh -o /tmp/hark-install.sh
# less /tmp/hark-install.sh
$ bash /tmp/hark-install.sh

One-liner (trust the release that published it): curl -fsSL https://hark.xk.io/install.sh | bash

Dogfood / stale CLI: if hark doctor reports install: stale|frozen, reinstall editable from your checkout: uv tool install -e . --force or ./install.sh.

Health — hark doctor

Run before first setup and whenever speech or Herdr looks wrong. Add --json for machine-readable output.

doctor
$ hark doctor
  • Fix red Herdr / tunnel / speech-key lines first.
  • Note the setup: line if incomplete or sessions are empty.
  • Doctor is discovery-oriented (credentials present); live API errors can still need a working provider.

First-run checklist

Use hark setup (non-interactive flags below) or let the skill ask one question at a time with hark ask / hark tts. Do not arm handsfree until sessions are written.

  1. Health hark doctor — clear install/Herdr/speech blockers.
  2. Herdr sessions Local, SSH, or mix → [[herdr.sessions]]. Never skip.
  3. Persona Feminine default: wake Iris (+ mercury/hark/herald), TTS eve. Masculine: Mercury, TTS leo. Or custom.
  4. TTS voice Any voice from the active provider catalog (samples when available).
  5. Wake backend Prefer Sherpa KWS when download works; Vosk if constrained; defer leaves stock vosk until you dogfood Sherpa.
  6. Model download If Sherpa: download script + uv sync --extra wake-sherpa. Fail-open to Vosk if download fails.
  7. Wake test Say hey iris / hey mercury (or your names). Optional: hark ambient --once.
  8. Setup flag Writes setup-complete.json (no secrets).
  9. Arm Continue with session profile + hark start / skill monitor loop.

CLI cheat

hark setup
$ hark setup --yes --persona feminine --wake-engine sherpa_kws --sessions local
# --force to re-run · --sessions local,work=ssh:box · --voice eve · --names iris,hark
# --skip-doctor · --skip-download

Marketing copy often says hey hark / hey herald — those are aliases. Product defaults after feminine setup are Iris (and related names), not brand-only wake.

Every /hark start

Even when setup is complete, the skill runs a structured startup interview (scope session-local vs Herdr, autonomy, role, mode) and applies it with hark session-profile set … --apply before ambient, watch, or monitor. Do not arm with silent defaults.

session-profile
$ hark session-profile show
$ hark session-profile set … --apply

If scope is herdr, confirm which sessions (local / SSH / mix). Details live in the skill and HERDR.md.

Workers — hark start

Mode A handsfree uses worker processes (Herdr watch + ambient wake), not the experimental harkd daemon. Do not run harkd alongside hark start.

lifecycle
$ hark start
$ hark start --status
$ hark stop
# --no-ambient · --no-watch · --force-watch · --session <id>
  • hark start watches one Herdr session id (default default) — configure multi-session in config; start flags are per invocation.
  • If ambient dies immediately, check wake model/deps with doctor — do not assume a phantom start succeeded.
  • Then follow with hark monitor (or the skill’s Monitor) for the handsfree loop.
